## 2. UX 영향 — 변경된 화면 5개

### 2.1 메인 피드 (가장 큰 변경)
- 이전: 별도 앱(워커 앱) 메인이 "오늘 일감"
- 신규: 단일 앱 → 사용자 역할 자동 감지(워커/구인자/욜드 자문가) → 메인 피드 분기 또는 통합 피드 + 필터
- **CPO 결정 필요**: "역할별 분기" vs "당근알바형 통합 피드 + 필터" — 다음 1주 와이어프레임 검토

### 2.2 공고 상세 (수수료 표기 제거)
- 이전: "시급 12,000원 · 일감 수수료 15% 차감 후 워커 수령 10,200원"
- 신규: "시급 12,000원 (전액 워커 수령)"
- 모든 시급·근무시간 표시 단순화

### 2.3 공고 등록 (구인자)
- 이전: 결제 정보 입력 → 에스크로 예치
- 신규: 결제 단계 제거. 단순 폼 입력 → 컨텐츠 QA 자동 검사 → 게시 (또는 운영자 검토)
- 컨텐츠 QA pending 상태 UX: "등록 검토 중 (평균 12분)" 표시

### 2.4 운영자 백오피스 (P0 화면)
- 이전: 구인자 승인 큐가 첫 화면
- 신규: **컨텐츠 QA 큐가 P0 첫 화면**. Claude API confidence 표시 + 1초 결정 UX. 분당 5~10건 처리 목표.
- 6개 스크린: QA 큐 / 신고 / 결제분쟁(축소) / 워커 ban / NSM 대시보드 / 감사로그

### 2.5 신규 화면: 욜드 Expert Advisory 진입
- 메인 피드와 별도 메뉴 "전문가 자문 찾기"
- 자문 카테고리 8종 (경영전략·재무·마케팅·인사·법무·해외진출·기술·ESG)
- 자문 프로필: 경력·시간당 단가·추천서·평점
- 자문 매칭 신청 → 일감 큐레이션 매니저 검토 → 매칭

## 3. QA 운영 부담 추정

### 3.1 자동 처리 비율 목표
- Tier 1 자동(키워드+URL 차단): 전체의 50%
- Tier 2 Claude API 자동 처리(confidence 0.85+): 추가 30%
- Tier 3 운영자 수동 검토: 15%
- Tier 4 외부 신고: 5%
- **자동 처리 합계 목표 80%+**

### 3.2 운영자 인력
- M3까지: 김연재 + CPO 임시 처리 (월 1,000건 미만)
- M3~M12: **운영자 1명 채용** (월 1,000~10,000건)
- M12~: 자동 처리 비율 따라 1~3명 (월 10,000~50,000건)

### 3.3 욜드 자문 큐레이션 별도
- M3~M6: **Tier 2 큐레이션 매니저 1명 추가 채용** (대기업 인사·영업 출신 우대)
- 욜드 풀 발굴 + 매칭 + 사후 평가 담당
- 인건비: 월 350~500만

## 4. CPO 검토 요청 사항 3건

### 검토 1 (즉시 — 다음 1주): 단일 앱 메인 피드 구조
- 옵션 A: 역할별 분기 (워커 → 일감 피드, 구인자 → 공고 관리, 욜드 → 자문 매칭)
- 옵션 B: 당근알바형 통합 피드 (역할 무관 단일 피드 + 필터)
- 옵션 C: 하이브리드 (메인 통합 + "내 역할" 별 빠른 진입)
- **CPO 권장 옵션 + 와이어프레임 1장 회신 요청**

### 검토 2 (다음 2주): QA 운영자 UX 사양
- 분당 5~10건 처리 목표 (Claude API confidence 표시 + 1초 결정)
- 키보드 단축키 우선 (마우스 이동 최소화): J/K 이동, A 승인, R 반려, F 신고
- 운영자 UX 와이어프레임 1장 + 결정 시간 측정 시뮬레이션 회신

### 검토 3 (다음 4주): 욜드 자문 프로필 UX
- 일반 워커 프로필과 시각적 차별화 필요 (프리미엄 톤)
- 경력 카드 + 추천서 + 자문 분야 8종 체크 + 시간당 단가 슬라이더
- CatchMe·Toptal·LinkedIn ProFinder 사례 비교 후 와이어프레임 회신
