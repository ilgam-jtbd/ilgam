# ADR-004 · 매칭 엔진 · 결제(단계화) · 알림 · API 계약 (v2.1 — Phase별 분기)

- **상태**: Accepted (v2.1)
- **일자**: 2026-04-28 (v1 04-23 / v2 04-24 / v2.1 04-28 단계화 — 변경 이력 §참조)
- **참여 에이전트**: Backend, CX, DB

## 매칭 엔진 (M1~M3) — 유지

**Postgres RPC(pl/pgSQL) + Edge Function 하이브리드.**

- 1차 필터(지역 반경·시급·시간대 overlap·차단 목록·`qa_status='approved'`) = PostGIS `ST_DWithin` + 인덱스 RPC 10~50ms
- 2차 랭킹(거리·시급·평점·노쇼율 가중합) = pl/pgSQL
- **Claude API**는 Edge Function(Deno)에서만:
  - **컨텐츠 QA 분류 (Tier 2, ADR-010)** — 신규 핵심 용도
  - 공고 요약·금칙어 검사·CS 트리아지
  - 건당 $0.003~0.01
- 매칭 랭킹에 LLM 미사용 (레이턴시 1.5~3초 · 비용 부적합)
- M3 임계: 워커 1,500명·일 매칭 800건 초과 시 Python worker(임베딩 유사도) 분리 검토

## 정산 모델 — Phase별 분기 (v2.1)

### Phase 0 — 직거래 (M0~M5 무료 검증)

1. 구인자 ↔ 워커 매칭 확정 (`matches`)
2. 워커 출근/퇴근 GPS+셀카 인증 (`shifts.clock_*`)
3. 구인자 승인 (`shifts.employer_approved_at`) — **여기서 종료**
4. 임금 지급은 **구인자가 워커에게 직접** (계좌이체·현금). 플랫폼 미개입.
5. 분쟁 시 채널톡 → `cx_tickets` → `(internal)/reports` 운영자 수동 개입

**법적 지위**: 통신판매중개업 신고. 플랫폼 자금 미개입 = 전자금융거래법 우려 없음.

### Phase 1 — PortOne 에스크로 + 10% 수수료 (M6+ 법인 설립)

법인 설립 + 10% 수수료 도입(인력사무소 오프라인 표준)과 동시에 정산 흐름 재구축:

1. **구인자 선결제**: PortOne 가상계좌 → 에스크로 예치 (총액 = 워커 임금 + 구인자 수수료 10%)
2. 워커 매칭 → 출근 → 퇴근 인증 → 구인자 승인(15분 무응답 자동승인)
3. **PortOne 송금 API**: 워커 임금 직접 송금 (T+0 VIP 가맹 또는 T+1 운영자금 5천만~1억으로 워커 당일 / 플랫폼 T+1)
4. 수수료 10% 플랫폼 회수 → `ad_payments` 가 아닌 신규 테이블 `commission_payments` 또는 `ad_payments` 재명명 검토 (M6 직전 별도 마이그레이션 PR)

**법적 지위**: 통신판매중개업 + PortOne 에스크로 = 전자금융거래법상 PG업 등록 회피 (M1 v1 패턴 부활).

**전환 시점**: M6 (2027.04) 법인 설립 + 10% 도입. 그 전까지 코드는 직거래만 작동.

### Phase 2 — Phase 1 + 광고/자문 멤버십 (M18~M30)

10% 수수료 위에 부가 매출:
- **광고 결제** (CPC ₩250 + B2B Premium 멤버십 ₩50~200만/월) — ADR-011 v1.1
- **자문 매칭 멤버십** (욜드 트랙 ₩50~200만/월) — ADR-011 v1.1

PortOne webhook 라우트 1개로 통합 (`payment-settle`이 정산·광고·자문 결제 모두 처리, 결제 종류는 메타데이터로 분기).

## 알림 어댑터 `notifyAdapter.send()` — 유지

- 시그니처: `send({templateId, channel, payload, userId})`
- 카카오 알림톡(Bizppurio) → 실패 시 SMS(알리고/LG U+) 폴백
- pg-boss 큐 + 지수 백오프 15s/1m/5m/15m
- **신규 템플릿** (피벗): `qa_pending`, `qa_approved`, `qa_rejected`, `qa_flagged_report` (ADR-010 연계)
- 매칭 확정·QA 결과 통보 등 critical은 우선순위 큐 분리

## Realtime vs 폴링 — 유지

- 구인자 앱: Supabase Realtime (지원·출근인증)
- 시니어 워커 앱: 15초 폴링 + Push Notification 기본
- 매칭 수락 화면 등 단기 포그라운드만 Realtime 한시 구독

## API 계약 — 유지

Supabase RPC 직통 + tRPC 얇은 레이어. 결제·정산 흐름 폐기로 **tRPC 보호 엔드포인트가 줄어듬** (Claude API + 광고 결제 placeholder만).

**가정**: MAU 1,050명(70%), 월 검증 공고 1,500건+ (피벗 NSM 베이스), Claude QA 호출 월 1만건 가정

| Vercel Pro | ~3만 | 동일 |
