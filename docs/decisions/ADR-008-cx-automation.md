# ADR-008 · CX 자동화 · 채널톡 · SMS Fallback · Claude 트리아지

- **상태**: Accepted
- **일자**: 2026-04-23
- **참여 에이전트**: CX, Backend

## 채널 역할 분리

| 채널 | M1 역할 |
|---|---|
| 채널톡 (Channel.io) | **메인 허브**. 봇 시나리오·Claude 트리아지·상담원 티켓 |
| 카카오채널 | 알림톡 아웃바운드 + "푸시 받은 김에 답장" 유입만 |
| 자체 인앱 문의 | "채널톡 열기" 단일 버튼으로 통합 |

근거: 카카오채널은 상담원 UI·라우팅 약함. 인앱 구현비 高 + 시니어 메뉴 이동 60~70% 이탈. 채널톡은 봇·SDK·Claude webhook 통합 검증.

## SMS 명령어 파싱 Fallback (2~3주 MVP)

흐름: `전용 번호 수신 → "문의 {본문}" 정규식 파싱 → 발신번호 유저 조회 → 채널톡 Open API User Chat 생성 + 메시지 주입 → "SMS 유입" 태그 라우팅`.

**리스크**: 양방향 SMS 사업자 계약(알뜰 LMS 수신 지원) · MMS·긴 문자 파편화 재조립.

**타당성**: SMS는 시니어 평생 습관 → 학습비 0. 알림톡 SMS 폴백 인프라 양방향 재활용.

## Claude API FAQ 자동답변

**2단 파이프라인**:
1. intent 분류(급여문의·공고오류·신고·계정·기타 5~7개) + confidence
2. 라우팅:
   - confidence ≥ 0.85 → FAQ 자동 송신
   - 0.60~0.85 → "이 답변이 맞나요?" 확인형 카드
   - < 0.60 또는 intent=신고·급여미지급 → 즉시 사람 이관

답변은 **사전 승인 FAQ 코퍼스 RAG 그라운딩**. 외부 생성 금지.

**피드백 루프**: 유저 "도움됨/안됨" 버튼 + 상담원 AI 답변 diff 수집 + 주 1회 오답 샘플 라벨링 → 프롬프트/FAQ 업데이트. 주간 정확도·이관율·CSAT 대시보드.

## 오프라인 지사 연동

디지털 해결 실패(3회 재질문 or 유저 "직접 가고 싶다" 발화) → 채널톡 봇이 현재 위치·등록 주소 기준 하버사인 정렬 → 영업시간(요일 JSON) 대조 → "지금 방문 가능 가까운 지사 3곳" 카드 → 알림톡 링크로 슬롯 예약.

**Handoff 프로토콜**: 지사 직원 태블릿(채널톡 팀 인박스) 4블록 카드 — 유저 요약 / 시도한 해결 / 원 대화 로그 / 필요 서류. 방문 후 "해결/미해결/재이관" 3버튼 마감 → 티켓 자동 동기화.

## 시니어용 CX 지표 3종 (재정의)

- **FRT**: "유저 첫 메시지 후 60초 내 사람 또는 봇 의미 있는 응답률", 목표 90% (시니어 타이핑 느림 반영)
- **CSAT**: 별점 대신 알림톡 "해결됐어요/아직이에요" 2지선다, 응답률 40% · 긍정률 85%
- **Self-serve resolution rate**: "7일 내 동일 intent 재문의 없는 종결" 재정의, 40~50%

## 악성·사기 신고 처리 (3트랙 격리)

### 가짜 공고
같은 사업자번호·전화에 중복 신고 2건 이상 OR Claude confidence ≥ 0.9 → 공고 즉시 shadow hide + 지원자 전원 주의 알림톡.

### 급여 미지급
접수 즉시 해당 사업주 신규 공고 게시 차단. 기존 매칭건 유지하되 "확인 중" 배지.

### 개인정보 요구
채팅 키워드 탐지(주민번호·계좌·카톡 ID 선요구) → 자동 경고 + 신고 큐 우선순위 상향.

**수동 리뷰 SLA**: 개인정보·사기 1차 2시간 / 최종 24시간. 가짜 공고 48시간. 급여 미지급 7영업일(노무·법률 파트너 연계). 모든 판정 사유 + 신고자·피신고자 알림톡 통지(투명성).
